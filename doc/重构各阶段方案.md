# A Dark Room — React 重构各阶段方案

## 总体策略：渐进式重构

从零搭建，逐模块实现。每个阶段聚焦一至两个技术点，完成后再进入下一阶段。

---

## 阶段 1：状态管理基础 🏗️

**目标**：搭建类型安全的全局状态层，替代原项目的 `State` + `$SM`。

### 任务拆分

| # | 任务 | 技术点 |
|---|------|--------|
| 1.1 | 定义核心 TypeScript 接口（`GameState`, `Stores`, `Game`, `Character` 等） | TypeScript 类型系统 |
| 1.2 | 实现 `GameProvider` + `useGameContext`（React Context + useReducer） | Context API |
| 1.3 | 实现类型安全的 `dispatch` actions（`SET`, `ADD`, `SET_M`, `ADD_M`） | useReducer 模式 |
| 1.4 | 添加 `stateUpdate` 订阅/通知机制（可用 `useEffect` 依赖或 EventEmitter） | 观察者模式 |
| 1.5 | 单元测试：状态读写、批量操作、边界值 | Vitest |

### 产出

- `src/state/types.ts` — 所有状态类型定义
- `src/state/GameContext.tsx` — Context Provider
- `src/state/reducer.ts` — Reducer + actions
- `src/state/__tests__/` — 测试

---

## 阶段 2：房间路由系统 🚪

**目标**：实现多场景切换，替代 `Engine.travelTo()` + `#locationSlider`。

### 任务拆分

| # | 任务 | 技术点 |
|---|------|--------|
| 2.1 | 实现场景路由：状态驱动条件渲染（`currentRoom` state） | React 条件渲染 |
| 2.2 | 实现 Header 标签栏组件 | 组件拆分 |
| 2.3 | 实现场景切换动画（CSS transition / framer-motion） | Tailwind + 动画 |
| 2.4 | 暗室场景基础 UI（标题、描述文字、火堆状态显示） | CSS Modules |
| 2.5 | 实现 `<Button>` 通用组件（冷却倒计时、消耗提示、禁用态） | 组件设计 |

### 产出

- `src/components/Header.tsx`
- `src/components/Button.tsx`
- `src/rooms/Room.tsx`
- `src/components/` 目录结构

## 阶段 3：资源系统 ⏱️

**目标**：实现资源定时增长 & 消耗逻辑。

### 任务拆分

| # | 任务 | 技术点 |
|---|------|--------|
| 3.1 | 资源显示组件（`StoresPanel`）：木材/兽皮/食物等实时展示 | Tailwind 布局 |
| 3.2 | 实现 `IncomeSystem`：定时收入 tick（`useEffect` + `setInterval`） | React 副作用 |
| 3.3 | 实现火堆机制：冷却定时器 + 添柴交互 | 定时器管理 |
| 3.4 | 实现建造者 NPC 阶段推进（状态变化 → 定时触发 → 通知） | 状态级联 |
| 3.5 | 实现建造/交易逻辑：消耗资源 → 获得建筑/物品 | 纯函数 reducer |

### 产出

- `src/systems/IncomeSystem.ts`
- `src/components/StoresPanel.tsx`
- `src/rooms/Room.tsx`（完整交互）

## 阶段 4：事件引擎 🎲

**目标**：实现声明式随机事件系统。

### 任务拆分

| # | 任务 | 技术点 |
|---|------|--------|
| 4.1 | 设计事件配置数据结构（`EventConfig` 类型） | 类型设计 |
| 4.2 | 实现事件池 + 随机调度器（权重、冷却、条件） | 算法 |
| 4.3 | 实现场景渲染引擎：递归渲染 `{ text, buttons, nextScene }` | 递归组件 |
| 4.4 | 实现 `<EventModal>` 弹窗组件（Portal 渲染） | React Portal |
| 4.5 | 实现状态触发事件（`onStateChange` → 匹配 → 触发） | 状态观察 |

### 产出

- `src/events/eventConfig.ts`
- `src/events/EventEngine.ts`
- `src/events/EventModal.tsx`

## 每阶段通用步骤

1. **分析**：对照原代码理解逻辑
2. **设计**：TypeScript 接口 + 组件树
3. **实现**：TDD 方式（先写测试）
4. **集成**：接入现有系统
5. **验证**：`pnpm build` + 手动测试
